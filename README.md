YÊU CẦU THỰC HÀNH <br>
9. Thực hành các thao tác Git trên dự án RE Shopper Web UI <br>
9.1. Thêm 1 người tham gia vào RE Shopper Repository mục Collaborators <br>
9.2. Live pair-working demo với người tham gia về việc xử lý xung đột code: <br>
9.2.1. Tạo nhánh mới (dev) từ nhánh main, thiết lập nhánh mặc định là dev <br>
9.2.2. Mỗi người tạo một nhánh mới (feature) từ nhánh dev (dev tạo từ main) <br>
9.2.3. Cả 2 cùng sửa tên dự án thành <Tên người tham gia> Shopper (tạo xung đột) <br>
9.2.4. Người A đẩy code lên GitHub trước, tạo Pull Request thứ nhất <br>
9.2.5. Gửi lên chat Zoom để nhờ Leader review, comment lên PR, approved PR <br>
9.2.6. Người A merge code vào dev, thông báo cho người B pull code mới về<br>
9.2.7. Người B pull code mới về sẽ gặp conflict với file code người B đã sửa<br>
9.2.8. Người B sẽ pair-reviewing với người A, xử lý xung đột code để giữ code cả 2<br>
9.2.9. Người B sẽ sửa tên dự án thành <tên người A + tên người B> Shopper<br>
9.2.10. Xử lý xong xung đột code, người B tạo Merge Conflict và đẩy PR lên GitHub<br>
=> Người B gửi link Pull Request lên chat Zoom nhờ Leader review (không bấm merge)<br>
Làm tới đây thì người A và người B gửi link PR lên nhóm Zalo để hoàn thành Case Study<br>
